---
title: createAgent
description: Create an AI agent with tools, memory, and streaming chat capabilities.
---

Creates an agent that combines a language model, tools, optional memory, and optional middleware/hooks into a single object that can stream responses back to clients.

```ts
import { createAgent } from "@zaikit/core";
```

## Usage

```ts
import { createAgent } from "@zaikit/core";
import { openai } from "@ai-sdk/openai";
import { createPostgresMemory } from "@zaikit/memory-postgres";

const memory = createPostgresMemory({
  connectionString: process.env.DATABASE_URL!,
});
await memory.initialize();

const agent = createAgent({
  model: openai("gpt-4o"),
  system: "You are a helpful assistant.",
  tools: { get_weather, search },
  memory,
});
```

## Options

`createAgent<T, C>(options: CreateAgentOptions<T, C>): Agent<T, C>`

| Option | Type | Required | Description |
| --- | --- | --- | --- |
| `name` | `string` | No | Agent name. Used as the default `functionId` for telemetry spans and appears in span names (e.g. `agent run: 'my-agent'`). See [Observability](/guides/observability). |
| `model` | `LanguageModel` | Yes | Any AI SDK compatible language model. |
| `system` | `string \| ((context: C) => string \| Promise<string>)` | No | System prompt sent with every request. Can be a function that receives the agent's context and returns the prompt — resolved once per request before the agent loop runs. |
| `tools` | `T` | No | Object mapping tool names to tool definitions. Values can be plain tools or `{ tool, mapContext }` entries when the agent has a context schema. |
| `context` | `z.ZodType<C>` | No | Zod schema defining the request-scoped context shape. When set, `agent.chat()` requires a `context` field, and tools with a matching `context` schema receive it in their execute function. See [Context](/concepts/context). |
| `memory` | `Memory` | No | A memory adapter for persisting threads and messages. Required if you call `agent.chat()`. |
| `middleware` | `Middleware[]` | No | Composable middleware that wraps the entire response. See [Middleware & Hooks](/concepts/middleware-and-hooks). |
| `prepareStep` | `PrepareStep<T, C>` | No | Hook that fires before each LLM call. Receives `{ steps, stepNumber, model, messages, context }` and returns per-step overrides (`activeTools`, `toolChoice`, `system`, `model`, `messages`). The `context` parameter is typed to the agent's context schema. See [Middleware & Hooks](/concepts/middleware-and-hooks#preparestep-per-step-overrides). |
| `onAfterStep` | `(ctx: AfterStepContext) => void \| Promise<void>` | No | Hook that fires after each LLM call. Receives `{ step, steps }` with the full `StepResult`. Throw to abort. |
| `onBeforeToolCall` | `(ctx: BeforeToolCallContext) => { input? } \| void \| Promise<...>` | No | Hook that fires before each tool execution. Return `{ input }` to override, throw to block. |
| `onAfterToolCall` | `(ctx: AfterToolCallContext) => { output? } \| void \| Promise<...>` | No | Hook that fires after each tool execution. Return `{ output }` to override. |
| `telemetry` | `boolean \| Partial<TelemetrySettings>` | No | Enable OpenTelemetry tracing. `true` enables with the agent `name` as `functionId`. Pass an object for fine-grained control. See [Observability](/guides/observability). |
| `onData` | `(part: WriteDataPart) => void` | No | Callback fired for every `writeData` call from any tool. See [Tool Data](/concepts/tool-data). |
| `onToolData` | `(event: ToolDataEvent) => void` | No | Callback fired for every `writeToolData` call. The event includes `toolName`, `toolCallId`, `type`, and `data`. See [Tool Data](/concepts/tool-data). |
| `onMetadata` | `(metadata: Record<string, unknown>) => void` | No | Callback fired for every `writeMetadata` call. Receives the metadata object that was written. |

## Return Type

`createAgent` returns an `Agent<T, C>` object:

```ts
type Agent<T extends ToolSet = ToolSet, C = undefined> = {
  tools: T;
  memory: Memory | undefined;
  model: LanguageModel;
  system: string | ((context: C) => string | Promise<string>) | undefined;
  contextSchema: Record<string, unknown> | undefined;
  stream(options: StreamOptions<C>): Promise<StreamResult>;
  chat(options: ChatOptions<C>): Promise<Response>;
  generate<OUTPUT extends z.ZodType = never>(
    options: GenerateOptions<C> & { output?: OUTPUT },
  ): Promise<GenerateResult<OUTPUT>>;
};
```

| Property | Type | Description |
| --- | --- | --- |
| `tools` | `T` | The tools object passed to the agent. Useful for codegen to inspect tool types. |
| `memory` | `Memory \| undefined` | The memory adapter, if one was provided. |
| `model` | `LanguageModel` | The language model instance. |
| `system` | `string \| ((context: C) => string \| Promise<string>) \| undefined` | The system prompt or function, if one was provided. |
| `contextSchema` | `Record<string, unknown> \| undefined` | JSON Schema representation of the context Zod type, if `context` was provided. Used by the sandbox UI to render context forms. |
| `stream` | `(options: StreamOptions<C>) => Promise<StreamResult>` | Pure agent execution. Returns a stream and a result promise. No memory or persistence -- the caller decides what to do with the output. |
| `chat` | `(options: ChatOptions<C>) => Promise<Response>` | Stream a response with persistence. Wraps `stream()` with memory, thread management, and title generation. Returns an HTTP `Response`. |
| `generate` | `<OUTPUT>(options) => Promise<GenerateResult<OUTPUT>>` | Run the agent and return structured results. Consumes the stream internally. No memory required. When `output` is provided, `result.output` is typed to match the schema. |

## `stream()`

Pure agent execution. Runs middleware and the core agent loop, returning a stream of UI message chunks and a promise that resolves to structured results when the stream completes. No memory, no persistence.

```ts
const { stream, result } = await agent.stream({
  messages: [{ id: "1", role: "user", parts: [{ type: "text", text: "Hello" }] }],
});

// Consume the stream (e.g. pipe to a client, or just drain it)
const reader = stream.getReader();
while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  // process chunks...
}

// Get structured results after stream completes
const agentResult = await result;
console.log(agentResult.text);        // final text
console.log(agentResult.steps);       // all StepResult objects
console.log(agentResult.usage);       // aggregated token usage
console.log(agentResult.finishReason); // "stop" | "tool-calls" | ...
```

### StreamOptions

```ts
type StreamOptions<C = undefined, T extends ToolSet = ToolSet> = ([C] extends [undefined]
  ? { context?: never }
  : { context: C }) & DataCallbacks<T> & {
  messages: UIMessage[];
  model?: LanguageModel;
  threadId?: string;
  maxSteps?: number;
  output?: z.ZodType;
  frontendTools?: FrontendToolDef[];
};
```

| Option | Type | Required | Description |
| --- | --- | --- | --- |
| `messages` | `UIMessage[]` | Yes | The conversation messages to send to the model. |
| `model` | `LanguageModel` | No | Per-call model override. Defaults to the agent's model. |
| `threadId` | `string` | No | Thread ID for middleware context. Auto-generated if not provided. |
| `maxSteps` | `number` | No | Maximum number of loop iterations. |
| `output` | `z.ZodType` | No | Zod schema for structured output. Parsed result is available on `AgentResult.output`. |
| `frontendTools` | `FrontendToolDef[]` | No | Frontend tool definitions to register dynamically. |
| `onData` | `(part: WriteDataPart) => void` | No | Callback fired for every `writeData` call from any tool during the stream. See [Tool Data](/concepts/tool-data). |
| `onToolData` | `(event: ToolDataEvent) => void` | No | Callback fired for every `writeToolData` call. See [Tool Data](/concepts/tool-data). |
| `onMetadata` | `(metadata: Record<string, unknown>) => void` | No | Callback fired for every `writeMetadata` call. |
| `context` | `C` | When agent has `context` | Request-scoped context passed to tools and hooks. |
| `userId` | `string` | No | User ID. Auto-injected as `metadata.userId` into telemetry when present. |
| `telemetry` | `boolean \| Partial<TelemetrySettings>` | No | Per-request telemetry override. `false` disables even if the agent has telemetry enabled. See [Observability](/guides/observability). |

### StreamResult

```ts
type StreamResult = {
  stream: ReadableStream<unknown>;
  result: Promise<AgentResult>;
};
```

| Property | Type | Description |
| --- | --- | --- |
| `stream` | `ReadableStream<unknown>` | UI message chunks. Must be consumed for the result promise to resolve. |
| `result` | `Promise<AgentResult>` | Resolves when the stream completes with structured results. |

### AgentResult

```ts
type AgentResult = {
  text: string;
  output: unknown;
  steps: StepResult<ToolSet>[];
  finishReason: string;
  usage: LanguageModelUsage;
};
```

| Property | Type | Description |
| --- | --- | --- |
| `text` | `string` | The text from the final step. |
| `output` | `unknown` | Parsed structured output when an `output` spec was provided. `undefined` otherwise. |
| `steps` | `StepResult[]` | All step results from the agent loop (AI SDK type). |
| `finishReason` | `string` | Why the model stopped (`"stop"`, `"tool-calls"`, etc.). |
| `usage` | `LanguageModelUsage` | Aggregated token usage across all steps (AI SDK type). |

## `generate()`

Run the agent and return structured results directly. Internally calls `stream()` and consumes the stream. Defaults `maxSteps` to 10.

```ts
// Simple prompt
const result = await agent.generate({ prompt: "What is 2 + 2?" });
console.log(result.text); // "4"

// Structured output — pass a Zod schema directly
const result = await agent.generate({
  prompt: "Rate this article",
  output: z.object({ title: z.string(), score: z.number() }),
});
console.log(result.output); // { title: "...", score: 85 }  — fully typed

// With messages
const result = await agent.generate({
  messages: [{ id: "1", role: "user", parts: [{ type: "text", text: "Hello" }] }],
});

// With tools and context
const result = await agent.generate({
  prompt: "Weather in NYC?",
  model: openai("gpt-4o-mini"), // per-call model override
  context: { userId: "user-123" },
});
```

The `output` option accepts a Zod schema. When provided, the model is constrained to JSON mode and the final step's text is parsed and validated against the schema. The result's `output` property is fully typed to match the schema.

When `output` is combined with tools, the agent runs the full tool loop and produces structured output on the final step — the model calls tools as needed, then responds with a JSON object matching the schema.

### GenerateOptions

```ts
type GenerateOptions<C = undefined, T extends ToolSet = ToolSet> = ([C] extends [undefined]
  ? { context?: never }
  : { context: C }) & DataCallbacks<T> & {
  model?: LanguageModel;
  maxSteps?: number;
  output?: z.ZodType;
  frontendTools?: FrontendToolDef[];
} & ({ prompt: string } | { messages: UIMessage[] });
```

| Option | Type | Required | Description |
| --- | --- | --- | --- |
| `prompt` | `string` | One of `prompt` or `messages` | A simple text prompt. Converted to a `UIMessage` internally. |
| `messages` | `UIMessage[]` | One of `prompt` or `messages` | Full conversation messages. |
| `model` | `LanguageModel` | No | Per-call model override. |
| `maxSteps` | `number` | No | Maximum loop iterations. Defaults to `10`. |
| `output` | `z.ZodType` | No | Zod schema for structured output. The model is constrained to JSON mode and the result is parsed and typed. |
| `frontendTools` | `FrontendToolDef[]` | No | Frontend tool definitions. |
| `onData` | `(part: WriteDataPart) => void` | No | Callback fired for every `writeData` call from any tool. See [Tool Data](/concepts/tool-data). |
| `onToolData` | `(event: ToolDataEvent) => void` | No | Callback fired for every `writeToolData` call. See [Tool Data](/concepts/tool-data). |
| `onMetadata` | `(metadata: Record<string, unknown>) => void` | No | Callback fired for every `writeMetadata` call. |
| `context` | `C` | When agent has `context` | Request-scoped context. |
| `userId` | `string` | No | User ID. Auto-injected as `metadata.userId` into telemetry when present. |
| `telemetry` | `boolean \| Partial<TelemetrySettings>` | No | Per-request telemetry override. See [Observability](/guides/observability). |

`generate()` does not support tool suspension. If a tool calls `suspend()`, `generate()` throws an error. Use `chat()` for tools that require human-in-the-loop interaction.

### GenerateResult

`GenerateResult` extends `AgentResult` with a typed `output` field. When `output` is provided, the result's `output` property is inferred from the Zod schema. Without `output`, it is `undefined`.

```ts
type GenerateResult<OUTPUT extends z.ZodType = never> = AgentResult & {
  output: [OUTPUT] extends [never] ? undefined : z.infer<OUTPUT>;
};
```

### Sub-agents

`generate()` is designed for invoking agents from within tools. Since tools are closures, sub-agents are accessed via closure scope -- no registry needed:

```ts
const subAgent = createAgent({
  model: openai("gpt-4o-mini"),
  system: "You summarize text concisely.",
});

const summarizeTool = createTool({
  description: "Summarize a document",
  inputSchema: z.object({ text: z.string() }),
  execute: async ({ input }) => {
    const result = await subAgent.generate({ prompt: input.text });
    return result.text;
  },
});

const mainAgent = createAgent({
  model: openai("gpt-4o"),
  tools: { summarize: summarizeTool },
  memory,
});
```

## ChatOptions

`ChatOptions<C>` is a discriminated union. The agent inspects the shape of the request body to determine the action. When the agent has a context schema (`C` is not `undefined`), all variants require a `context: C` field:

```ts
type ChatOptions<C = undefined> =
  ([C] extends [undefined] ? { context?: never } : { context: C }) &
  { threadId: string; userId?: string; frontendTools?: FrontendToolDef[]; telemetry?: boolean | Partial<TelemetrySettings> } &
  (
    | { message: UIMessage }
    | { resume: { toolCallId: string; data: unknown } }
    | { toolOutputs: { toolCallId: string; output: unknown }[] }
  );
```

### Message path

When the body contains `message`, the agent:

1. Creates the thread if it does not exist. If `userId` is provided, it is stored on the new thread.
2. Persists the user message via `memory.addMessage`.
3. Loads the full conversation via `memory.getMessages`.
4. Fires off a background title generation request (fire-and-forget).
5. Streams the LLM response back, persisting the assistant message on finish.

### Resume path

When the body contains `resume`, the agent:

1. Finds the suspended message by scanning for a `data-tool-suspend` part matching `resume.toolCallId`.
2. Re-executes the tool with `resumeData` passed via `AsyncLocalStorage`.
3. Updates the message in-place with the tool output.
4. If other tools in the same message are still suspended, returns `204 No Content` -- the client should re-fetch messages.
5. Otherwise, continues the LLM with the updated conversation and streams the follow-up.

### Tool outputs path

When the body contains `toolOutputs`, the agent handles frontend tool results:

1. For each `{ toolCallId, output }` entry, finds the corresponding tool call and writes the output.
2. Persists the updated message.
3. If other tools are still pending, returns `204 No Content`.
4. Otherwise, continues the LLM loop and streams the follow-up.

### `frontendTools`

All three variants accept an optional `frontendTools` array. When provided, the agent dynamically registers these as tools that the LLM can call but whose execution is deferred to the client. See `FrontendToolDef`:

```ts
type FrontendToolDef = {
  name: string;
  description: string;
  parameters: Record<string, unknown>;
};
```

## Middleware & Hooks

The agent supports three levels of interception. See [Middleware & Hooks](/concepts/middleware-and-hooks) for full details and examples.

```ts
import { createAgent, type Middleware } from "@zaikit/core";

const logging: Middleware = (ctx, next) => {
  console.log(`Processing ${ctx.messages.length} messages`);
  return next();
};

const agent = createAgent({
  model: openai("gpt-4o"),
  tools: { confirm_purchase },
  memory,
  middleware: [logging],
  onAfterStep: ({ steps }) => {
    if (steps.length >= 5) throw new Error("Max steps reached");
  },
});
```

## Wiring to an HTTP Endpoint

`agent.chat()` returns a standard `Response` with a streaming body, so it works with any HTTP framework:

```ts
// Hono
app.post("/api/chat", async (c) => {
  const body = await c.req.json();
  return agent.chat(body);
});

// Express (with a streaming adapter)
app.post("/api/chat", async (req, res) => {
  const response = await agent.chat(req.body);
  // Pipe response.body to res
});
```
