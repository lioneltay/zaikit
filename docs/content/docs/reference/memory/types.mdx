The `@zaikit/memory` package defines the `Memory` and `Thread` types that all memory adapters implement. Install it directly when building a custom adapter, or use it through `@zaikit/core` which re-exports these types.

```bash
pnpm add @zaikit/memory
```

```ts
import type { Memory, Thread } from "@zaikit/memory";

// Also re-exported from @zaikit/core for convenience
import type { Memory, Thread } from "@zaikit/core";
```

## Thread Type

```ts
type Thread = {
  id: string;
  title: string | null;
  userId: string | null;
  createdAt: Date;
  updatedAt: Date;
};
```

| Property | Type | Description |
| --- | --- | --- |
| `id` | `string` | Unique identifier for the thread. Typically a UUID. |
| `title` | `string \| null` | Thread title. `null` until auto-generated by the agent after the first message. |
| `userId` | `string \| null` | Optional user identifier. Use to associate threads with a user, service, or org. |
| `createdAt` | `Date` | When the thread was created. |
| `updatedAt` | `Date` | When the thread was last modified (message added, title updated, etc.). |

## Memory Interface

```ts
type GetMessagesOptions = {
  limit?: number;
};

type Memory = {
  initialize(): Promise<void>;
  close(): Promise<void>;

  createThread(id: string, title?: string, userId?: string): Promise<Thread>;
  getThread(id: string): Promise<Thread | null>;
  listThreads(opts?: { userId?: string }): Promise<Thread[]>;
  deleteThread(id: string): Promise<void>;
  updateThread(id: string, updates: { title?: string; userId?: string }): Promise<Thread>;

  getMessages(threadId: string, options?: GetMessagesOptions): Promise<UIMessage[]>;
  addMessage(threadId: string, message: UIMessage): Promise<void>;
  updateMessage(
    threadId: string,
    messageId: string,
    updates: { parts: UIMessage["parts"] }
  ): Promise<void>;
};
```

`UIMessage` is the AI SDK's message type from the `ai` package.

## Method Reference

### Lifecycle Methods

| Method | Signature | Description |
| --- | --- | --- |
| `initialize` | `() => Promise<void>` | Create tables, indexes, or other storage structures. Idempotent — safe to call on every startup. |
| `close` | `() => Promise<void>` | Close database connections and release resources. |

### Thread Methods

| Method | Signature | Description |
| --- | --- | --- |
| `createThread` | `(id: string, title?: string, userId?: string) => Promise<Thread>` | Create a new thread with the given ID, optional title, and optional user. |
| `getThread` | `(id: string) => Promise<Thread \| null>` | Retrieve a thread by ID. Returns `null` if not found. |
| `listThreads` | `(opts?: { userId?: string }) => Promise<Thread[]>` | List threads, optionally filtered by `userId`. Ordered by `updatedAt` descending. |
| `deleteThread` | `(id: string) => Promise<void>` | Delete a thread and all its messages. |
| `updateThread` | `(id: string, updates: { title?: string; userId?: string }) => Promise<Thread>` | Update thread fields. Uses partial updates -- only provided fields are changed. |

### Message Methods

| Method | Signature | Description |
| --- | --- | --- |
| `getMessages` | `(threadId: string, options?: GetMessagesOptions) => Promise<UIMessage[]>` | Get all messages in a thread, ordered by creation time. Pass `{ limit: N }` to return only the most recent N messages. |
| `addMessage` | `(threadId: string, message: UIMessage) => Promise<void>` | Add a message to a thread. Implementations should use upsert semantics (insert or update if the message ID already exists). |
| `updateMessage` | `(threadId: string, messageId: string, updates: { parts: UIMessage["parts"] }) => Promise<void>` | Update the parts of an existing message. Used by the agent during resume to update suspended tool outputs in-place. |

## Building a Custom Adapter

Implement the `Memory` interface for any storage backend:

```ts
import type { Memory, Thread } from "@zaikit/memory";
import type { UIMessage } from "ai";

export function createMyMemory(): Memory {
  return {
    async initialize() {
      // Create tables/indexes (idempotent)
    },

    async close() {
      // Close database connections
    },

    async createThread(id, title, userId) {
      // Insert into your storage
      // Return the created Thread
    },

    async getThread(id) {
      // Look up by ID
      // Return Thread or null
    },

    async listThreads(opts) {
      // Return threads, optionally filtered by opts.userId
      // Ordered by updatedAt descending
    },

    async deleteThread(id) {
      // Delete the thread and cascade-delete its messages
    },

    async updateThread(id, updates) {
      // Partial update -- only modify provided fields
      // Return the updated Thread
    },

    async getMessages(threadId, options) {
      // Return messages ordered by creation time
      // If options.limit is set, return only the most recent N messages
    },

    async addMessage(threadId, message) {
      // Upsert: insert or update if message.id already exists
    },

    async updateMessage(threadId, messageId, updates) {
      // Update the parts array of an existing message
    },
  };
}
```

### Key Implementation Notes

- **Upsert semantics**: `addMessage` should handle the case where a message with the same ID already exists. The agent calls `addMessage` during streaming, and the `onFinish` callback may write back an updated version of the same message.
- **Cascade delete**: `deleteThread` should also remove all messages belonging to that thread.
- **Partial updates**: `updateThread` should only modify fields that are present in the `updates` object. Use `COALESCE` (SQL) or equivalent logic.
- **Ordering**: `getMessages` must return messages in chronological order. `listThreads` should return threads in reverse-chronological order by `updatedAt`.

## Conformance Tests

`@zaikit/memory` ships a conformance test suite you can run against your custom adapter to verify correctness:

```ts
import { memoryConformanceTests } from "@zaikit/memory/test";
import { createMyMemory } from "./index";

memoryConformanceTests({
  create: () => createMyMemory(),
});
```

The function accepts an options object:

| Option | Type | Required | Description |
| --- | --- | --- | --- |
| `create` | `(ctx: TContext) => Memory \| Promise<Memory>` | Yes | Factory that returns a fresh `Memory` instance. The framework calls `initialize` and cleans up between tests automatically. |
| `start` | `() => TContext \| Promise<TContext>` | No | Start backing infrastructure (e.g. a test container). Runs once before all tests. The returned context is passed to `create`. |
| `stop` | `(ctx: TContext) => void \| Promise<void>` | No | Tear down infrastructure. Runs once after all tests. |

When `start` is provided, the suite also runs persistence tests that verify data survives across separate `Memory` instances connected to the same backing store.

Example with a test container:

```ts
import { MongoDBContainer, type StartedMongoDBContainer } from "@testcontainers/mongodb";
import { memoryConformanceTests } from "@zaikit/memory/test";
import { createMongoMemory } from "@zaikit/memory-mongo";

memoryConformanceTests<StartedMongoDBContainer>({
  start: () => new MongoDBContainer("mongo:7").start(),
  stop: (container) => container.stop(),
  create: (container) =>
    createMongoMemory({
      url: `${container.getConnectionString()}?directConnection=true`,
    }),
});
```

The suite covers all thread and message operations including edge cases like upsert semantics, cascade deletes, ordering, and `getMessages` limit.
