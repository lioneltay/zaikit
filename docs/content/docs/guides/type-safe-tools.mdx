## The Problem

When you use `useToolRenderer`, the render callback receives `ToolRenderProps` with generic types:

```ts
type ToolRenderProps = {
  toolCallId: string;
  toolName: string;
  state: "call" | "suspended" | "result" | "error";
  args: Record<string, unknown>;       // Not typed to your tool's input
  suspendPayload: unknown;              // Not typed to your suspend schema
  result: unknown;                      // Not typed
  error: string | undefined;            // Not typed
  resume: (data: unknown) => void;      // Accepts anything
};
```

This means you have to cast or assert types manually in every tool component:

```tsx
// Without codegen -- lots of type assertions
function BookFlightTool(props: ToolRenderProps) {
  const args = props.args as { destination: string; date: string };
  const flights = (props.suspendPayload as { flights: Flight[] })?.flights;
  // ...
}
```

## The Solution

`@zaikit/codegen-react` reads your agent's tool type information at build time and generates typed props aliases for each tool. Your components get full type safety with zero manual casting.

```tsx
// With codegen -- fully typed
import type { BookFlightToolProps } from "./tools.generated";

function BookFlightTool(props: BookFlightToolProps) {
  props.args.destination;      // string -- autocomplete works
  props.suspendPayload?.flights; // { id: string; airline: string; ... }[]
  props.resume({
    selectedFlightId: "QF42",
    seatPreference: "window",  // "window" | "aisle" | "middle"
  });
}
```

## Setup

### 1. Install

```bash
pnpm add -D @zaikit/codegen-react
```

### 2. Add a Script

```json title="package.json"
{
  "scripts": {
    "generate": "zaikit-codegen-react --agent ./src/agent/index.ts --output ./src/tools/tools.generated.ts"
  }
}
```

### 3. Run It

```bash
pnpm generate
```

## CLI Usage

```bash
zaikit-codegen-react \
  --agent ./src/agent/index.ts \
  --output ./src/tools/tools.generated.ts \
  --export agent \
  --tsconfig ./tsconfig.json
```

| Flag | Required | Default | Description |
|---|---|---|---|
| `--agent` | Yes | -- | Path to the file that exports your agent |
| `--output` | Yes | -- | Path where the generated file will be written |
| `--export` | No | `agent` | Name of the export to read (e.g. `export const agent = createAgent(...)`) |
| `--tsconfig` | No | Auto-detected | Path to `tsconfig.json` (walks up from the agent file if not specified) |

## What Gets Generated

Given an agent with `get_weather` and `book_flight` tools, the generated file looks like:

```ts title="tools.generated.ts"
// AUTO-GENERATED by @zaikit/codegen-react — DO NOT EDIT

import type { ReactNode } from "react";
import type { ToolRenderProps, ToolRenderFn } from "@zaikit/react";
import { useToolRenderer as _useToolRenderer } from "@zaikit/react";

// ─── Types ───

export type GetWeatherInput = { location: string };
export type GetWeatherOutput = {
  location: string;
  temperature: number;
  condition: string;
  humidity: number;
  windSpeed: number;
};

export type BookFlightInput = { destination: string; date: string };
export type BookFlightOutput = { confirmation: string; status: string; flight: { id: string; airline: string; price: number; departure: string }; seat: string };
export type BookFlightSuspend = { flights: { id: string; airline: string; price: number; departure: string }[] };
export type BookFlightResume = { selectedFlightId: string; seatPreference: "window" | "aisle" | "middle" };

// ─── Typed Props ───

export type GetWeatherToolProps = ToolRenderProps<GetWeatherInput>;

export type BookFlightToolProps = ToolRenderProps<BookFlightInput, BookFlightSuspend, BookFlightResume>;

// ─── Typed useToolRenderer ───

type ToolPropsMap = {
  get_weather: GetWeatherToolProps;
  book_flight: BookFlightToolProps;
};

export function useToolRenderer<T extends keyof ToolPropsMap>(
  name: T,
  render: (props: ToolPropsMap[T]) => ReactNode
): void;
export function useToolRenderer(name: string, render: ToolRenderFn): void;
export function useToolRenderer(name: string, render: ToolRenderFn): void {
  _useToolRenderer(name, render);
}

// ─── Tool Name Union ───

export type ToolName =
  | "get_weather"
  | "book_flight";
```

Each tool produces:

| Generated type | Description |
|---|---|
| `{PascalName}Input` | The tool's input type (from `inputSchema`) |
| `{PascalName}Output` | The tool's output type (from `outputSchema` or the inferred return type) |
| `{PascalName}Suspend` | The suspend payload type (only for suspendable tools) |
| `{PascalName}Resume` | The resume data type (only for suspendable tools) |
| `{PascalName}ToolProps` | A `ToolRenderProps<Input, Suspend, Resume>` alias ready to use as component props |

The generated `useToolRenderer` function is a typed wrapper. When you pass a known tool name, the render callback gets the corresponding typed props:

```tsx
import { useToolRenderer } from "./tools.generated";

// props is BookFlightToolProps -- fully typed
useToolRenderer("book_flight", (props) => <BookFlightTool {...props} />);
```

A `ToolName` union type is also generated for use in generic utility functions.

## Using Generated Types in Components

Import the generated props type and use it directly:

```tsx title="src/tools/BookFlightTool.tsx"
import { useState } from "react";
import type { BookFlightToolProps } from "./tools.generated";

export function BookFlightTool(props: BookFlightToolProps) {
  const flights = props.suspendPayload?.flights ?? [];
  const [selectedId, setSelectedId] = useState<string | null>(null);
  const [seat, setSeat] = useState<"window" | "aisle" | "middle">("window");

  if (props.state === "result") {
    return <p>Flight booked.</p>;
  }

  return (
    <div>
      <h4>Select a Flight</h4>
      {flights.map((f) => (
        <div key={f.id} onClick={() => setSelectedId(f.id)}>
          {f.airline} -- {f.id} (${f.price})
        </div>
      ))}
      <select value={seat} onChange={(e) => setSeat(e.target.value as any)}>
        <option value="window">Window</option>
        <option value="aisle">Aisle</option>
        <option value="middle">Middle</option>
      </select>
      <button
        disabled={!selectedId}
        onClick={() => {
          if (!selectedId) return;
          props.resume({ selectedFlightId: selectedId, seatPreference: seat });
        }}
      >
        Book
      </button>
    </div>
  );
}
```

### Registering Renderers

```tsx title="src/tools/index.tsx"
import { useToolRenderer } from "./tools.generated";
import { BookFlightTool } from "./BookFlightTool";

export function ToolRenderers() {
  useToolRenderer("book_flight", (props) => <BookFlightTool {...props} />);

  // Wildcard fallback -- uses the untyped ToolRenderProps
  useToolRenderer("*", (props) => <DefaultTool {...props} />);

  return null;
}
```

## How It Works

### The `__toolTypes` Phantom Brand

`createTool` returns a `ZaikitTool` type that carries a phantom `__toolTypes` property:

```ts
type ZaikitTool<INPUT, OUTPUT, SUSPEND, RESUME, CONTEXT> = Tool<INPUT, OUTPUT> & {
  readonly __toolTypes: {
    readonly input: INPUT;
    readonly output: Exclude<OUTPUT, SuspendResult<any>>;
    readonly suspend: SUSPEND;
    readonly resume: RESUME;
    readonly context: CONTEXT;
  };
  readonly __meta?: ToolMeta;
};
```

This property exists only in the type system -- it has no runtime cost and no runtime value. The codegen tool uses `ts-morph` to read this type information from your compiled TypeScript project.

### Extraction Pipeline

1. `ts-morph` loads your project using the `tsconfig.json`
2. It finds the named export (default: `agent`) in your agent file
3. It reads the `tools` property on the agent's type
4. For each tool, it reads the `__toolTypes` brand to extract `input`, `output`, `suspend`, `resume`, and `context` types
5. These are rendered as TypeScript type aliases in the output file

Tools that were not created with `createTool` (and thus lack the `__toolTypes` brand) are skipped with a warning.

## CI Integration

Add the generate step before your type-check in CI to ensure generated types are up-to-date:

```json title="package.json"
{
  "scripts": {
    "generate": "zaikit-codegen-react --agent ./src/agent/index.ts --output ./src/tools/tools.generated.ts",
    "typecheck": "tsc --noEmit",
    "ci": "pnpm generate && pnpm typecheck"
  }
}
```

You can either commit the generated file to version control or regenerate it in CI. If you commit it, add a CI check that verifies the file is up-to-date:

```bash
pnpm generate
git diff --exit-code src/tools/tools.generated.ts
```

If the diff is non-empty, the agent's tools have changed but the generated types were not regenerated.

## Tips

- Run `pnpm generate` whenever you add, remove, or change a tool's schemas
- The generated file should not be edited by hand -- it will be overwritten on the next run
- If `tsconfig.json` cannot be auto-detected (e.g., in a monorepo with multiple configs), use the `--tsconfig` flag explicitly
- The `--export` flag defaults to `agent`, matching `export const agent = createAgent(...)`. If your export is named differently, specify it: `--export myAgent`
